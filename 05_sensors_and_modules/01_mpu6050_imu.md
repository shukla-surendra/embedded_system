# 01 — MPU-6050 IMU: Accelerometer + Gyroscope

## What Is It?

The **MPU-6050** is a 6-axis Inertial Measurement Unit (IMU) made by InvenSense.
"6-axis" means it measures:

- **3-axis Accelerometer** — measures linear acceleration (m/s²) on X, Y, Z
- **3-axis Gyroscope** — measures angular velocity (°/s) on X, Y, Z

For a drone, this is the most critical sensor. The flight controller reads it hundreds of times per second to know how the drone is tilting and spinning, then fires the motors to correct it.

---

## The Physics in Plain Language

### Accelerometer
Imagine a ball inside a box. When you tilt the box, gravity pulls the ball toward one wall.
The accelerometer measures that force. At rest flat on a table:
- X = 0 g
- Y = 0 g
- Z = 1 g (gravity pointing down)

When you tilt it 45°, the X and Z readings share the gravity vector. From these numbers you can calculate **pitch** and **roll** angles.

**Limitation:** The accelerometer is noisy during vibration (like a flying drone). It's good for slow angles but bad at fast rotations.

### Gyroscope
The gyroscope measures *how fast* the sensor is rotating (degrees per second). Integrate that over time → you get angle change.

**Limitation:** It drifts. Tiny errors accumulate over time, so after a few minutes the reported angle is wrong.

### The Solution: Sensor Fusion
Combine both:
- Gyroscope gives accurate fast motion
- Accelerometer gives a stable long-term reference

Algorithms like **Complementary Filter** or **Kalman Filter** blend them. Most flight controllers use this approach.

## Hardware Overview

```
MPU-6050 Module Pins:

  VCC  ── 3.3V or 5V (module has onboard regulator)
  GND  ── Ground
  SCL  ── I2C Clock
  SDA  ── I2C Data
  XDA  ── External I2C data (for aux sensors, usually unused)
  XCL  ── External I2C clock (usually unused)
  AD0  ── I2C address select (LOW = 0x68, HIGH = 0x69)
  INT  ── Interrupt pin (fires when new data is ready)
```

I2C default address: **0x68** (AD0 pin LOW / floating)

---

## Wiring to ESP32

| MPU-6050 Pin | ESP32 Pin     | Notes                    |
|--------------|---------------|--------------------------|
| VCC          | 3.3V          | 3.3V recommended         |
| GND          | GND           |                          |
| SCL          | GPIO 22       | Default I2C SCL on ESP32 |
| SDA          | GPIO 21       | Default I2C SDA on ESP32 |
| AD0          | GND           | Sets address to 0x68     |
| INT          | Not connected | Optional for interrupts  |

> Use short wires (< 20 cm). I2C is not designed for long cable runs.

## Code Example 1 — Basic Raw Data

```cpp
#include <Wire.h>
#include <MPU6050_light.h>

MPU6050 mpu(Wire);

void setup() {
  Serial.begin(115200);
  Wire.begin();           // SDA=21, SCL=22 by default on ESP32

  byte status = mpu.begin();
  if (status != 0) {
    Serial.print("MPU6050 init failed, error code: ");
    Serial.println(status);
    while (true);
  }

  Serial.println("Calibrating... keep sensor still for 3 seconds");
  delay(1000);
  mpu.calcOffsets();      // auto-calibrate gyro and accelerometer
  Serial.println("Done!");
}

void loop() {
  mpu.update();           // fetch new data from sensor

  Serial.print("Pitch: "); Serial.print(mpu.getAngleX());
  Serial.print("°  Roll: "); Serial.print(mpu.getAngleY());
  Serial.print("°  Yaw: "); Serial.print(mpu.getAngleZ());
  Serial.println("°");

  delay(100);
}
```

**Expected Serial output (sensor flat, still):**
```
Pitch: 0.12°  Roll: -0.08°  Yaw: 0.00°
```

Tilt the sensor — watch Pitch or Roll change.

## Code Example 2 — Raw Accelerometer + Gyro Values

```cpp
#include <Wire.h>
#include <MPU6050_light.h>

MPU6050 mpu(Wire);

void setup() {
  Serial.begin(115200);
  Wire.begin();
  mpu.begin();
  mpu.calcOffsets();
}

void loop() {
  mpu.update();

  // Acceleration in g (1g = 9.8 m/s²)
  Serial.print("AccX: "); Serial.print(mpu.getAccX());
  Serial.print("  AccY: "); Serial.print(mpu.getAccY());
  Serial.print("  AccZ: "); Serial.println(mpu.getAccZ());

  // Angular velocity in degrees/second
  Serial.print("GyroX: "); Serial.print(mpu.getGyroX());
  Serial.print("  GyroY: "); Serial.print(mpu.getGyroY());
  Serial.print("  GyroZ: "); Serial.println(mpu.getGyroZ());

  Serial.println("---");
  delay(200);
}
```

## Code Example 3 — Simple Complementary Filter (manual, no library math)

This shows the actual math used in flight controllers.

```cpp
#include <Wire.h>
#include <MPU6050_light.h>

MPU6050 mpu(Wire);

float pitch = 0, roll = 0;
unsigned long lastTime = 0;
const float ALPHA = 0.96;   // trust gyro 96%, accel 4%

void setup() {
  Serial.begin(115200);
  Wire.begin();
  mpu.begin();
  mpu.calcOffsets();
  lastTime = millis();
}

void loop() {
  mpu.update();

  float dt = (millis() - lastTime) / 1000.0;  // seconds since last loop
  lastTime = millis();

  // Angle from accelerometer (absolute but noisy)
  float accelPitch = atan2(mpu.getAccY(), mpu.getAccZ()) * 180.0 / PI;
  float accelRoll  = atan2(mpu.getAccX(), mpu.getAccZ()) * 180.0 / PI;

  // Complementary filter: blend gyro integration with accel correction
  pitch = ALPHA * (pitch + mpu.getGyroX() * dt) + (1 - ALPHA) * accelPitch;
  roll  = ALPHA * (roll  + mpu.getGyroY() * dt) + (1 - ALPHA) * accelRoll;

  Serial.print("Pitch: "); Serial.print(pitch);
  Serial.print("°  Roll: "); Serial.println(roll);

  delay(10);  // 100 Hz loop rate
}
```

## Understanding the Numbers

| Situation                     | AccZ  | AccX  | Pitch  |
|-------------------------------|-------|-------|--------|
| Flat on table                 | ~1.0g | ~0.0g | ~0°    |
| Tilted 45° nose up            | ~0.7g | ~0.7g | ~45°   |
| Vertical (nose pointing up)   | ~0.0g | ~1.0g | ~90°   |
| Upside down                   | ~-1g  | ~0.0g | ~180°  |

## Common Problems

| Problem                          | Likely Cause                          | Fix                                      |
|----------------------------------|---------------------------------------|------------------------------------------|
| "MPU6050 init failed"            | Wrong wiring or I2C address           | Check SDA/SCL pins, check AD0 is grounded|
| Angles drift slowly              | Gyro drift (normal)                   | Use complementary/Kalman filter          |
| Angles jump wildly               | Vibration or loose wires              | Better wire connections, shorter cables  |
| All zeros                        | VCC not connected or wrong voltage    | Check power, use 3.3V                    |
| calibrating takes too long       | Sensor was moving during calibration  | Hold it perfectly still for 3 seconds   |

## How This Applies to Drone Flight

On a drone, the MPU-6050 is read ~400–1000 times per second.

```
Motor speed → Drone tilts → MPU detects angle error
→ PID algorithm → Correct motor speeds → Drone levels out
```

This feedback loop runs so fast that the drone self-corrects faster than you can blink. Without the IMU, a quadcopter is unflyable — it would flip in under a second.
